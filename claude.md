# Project Rules

- Always write a test for any new function.
- Use clear, descriptive variable names — no single letters except loop counters like i.
- Never delete anything inside the notes/ folder.
- Ask before adding any new external library/package. 
- Always double-check that output is actually correct, not just that the program ran without errors — a program can run cleanly and still produce a wrong result
- Always ensure the program can handle invalid inputs gracefully, rather than crashing or silently doing nothing.
